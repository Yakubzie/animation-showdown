Implementation Details

Code examples and technical implementation differences

ScrollTrigger Implementation

// GSAP ScrollTrigger
gsap.from(".element", {
  y: 100,
  opacity: 0,
  duration: 1,
  ease: "power3.out",
  scrollTrigger: {
    trigger: ".element",
    start: "top 80%",
    end: "bottom 20%",
    toggleActions: "play none none reverse"
  }
});
// Vanilla JS Intersection Observer
const observer = new IntersectionObserver(
  (entries) => {
    entries.forEach(entry => {
      if (entry.isIntersecting) {
        entry.target.classList.add('animate-in');
      }
    });
  },
  { threshold: 0.1, rootMargin: '0px 0px -100px 0px' }
);

observer.observe(document.querySelector('.element'));

Stagger Animations

// GSAP Stagger
gsap.from(".stagger-item", {
  y: 50,
  opacity: 0,
  duration: 0.8,
  stagger: 0.1,
  ease: "power2.out"
});
// Vanilla JS Stagger
const items = document.querySelectorAll('.stagger-item');
items.forEach((item, index) => {
  setTimeout(() => {
    item.classList.add('animate-in');
  }, index * 100);
});

SVG Path Animation

// GSAP SVG Path
const path = document.querySelector('.path');
const length = path.getTotalLength();

gsap.set(path, {
  strokeDasharray: length,
  strokeDashoffset: length
});

gsap.to(path, {
  strokeDashoffset: 0,
  duration: 2,
  ease: "power2.out"
});
// Vanilla JS SVG Path
const path = document.querySelector('.path');
const length = path.getTotalLength();

path.style.strokeDasharray = length;
path.style.strokeDashoffset = length;

path.animate(
  { strokeDashoffset: 0 },
  { duration: 2000, easing: 'ease-out' }
);

Use Case Recommendations

When to choose GSAP vs Vanilla JS based on project requirements

Choose GSAP When:

  • Complex animation sequences with multiple elements
  • Advanced scroll-based animations with precise control
  • Professional projects requiring polished animations
  • Cross-browser compatibility is critical
  • Team has GSAP experience or time to learn
  • Performance optimization is a priority

Choose Vanilla JS When:

  • Simple animations with minimal complexity
  • Bundle size optimization is critical
  • No external dependencies preferred
  • Modern browsers only (no IE support needed)
  • Team prefers native browser APIs
  • Budget constraints limit commercial licenses
